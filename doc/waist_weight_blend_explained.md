# XPS→MMD 转换：腰部权重混合原理与问题分析

> 记录：2026-03-24
> 问题：腰骨运动时网格变形不平滑（硬切割/裂口感）
> 对应修复：`bone_operator.py` `_weight_cleanup_leg_torso_conflict` 阈值化 + `LEG_FK_TO_D` 映射表

---

## 一、MMD 腰部骨骼层级关系

MMD 的腿部有两套骨骼：

```
下半身
 └── 腰キャンセル.L
      └── 足D.L          ← D系变形骨（真正控制网格）
           └── ひざD.L
                └── 足首D.L

下半身
 └── 左足（FK骨）         ← use_deform=False，不变形网格
      └── 左ひざ
           └── 左足首
```

`足D.L` 通过 **付与（additional_transform）** 跟随 `左足` 的旋转。

---

## 二、为什么腰臀区需要两个骨骼同时有权重？

想象一下做"抬腿"动作：

```
动作前（T-Pose）          动作后（抬左腿）
─────────────             ─────────────
   腰部网格                  腰部网格
   ┌───────┐                ┌───────┐
   │ 下半身│  ←髋骨          │ 下半身│  ←不动
   ├───┬───┤                ├────/──┤  ←腹股沟顶点
   │   │   │                │   /   │   要"平滑跟随"
   │ 足D.L │  ←大腿          │  /    │
   │   │   │                │/足D.L │  ←抬起
```

**腹股沟/大腿根部**的顶点不能只跟下半身（髋骨动了腿纹丝不动），也不能只跟足D（腿抬起时腹部被拉走）。

正确做法：这些顶点**同时拥有两个骨骼的权重**，比如：
- 下半身权重 = 0.7，足D.L权重 = 0.3 → 顶点走 70% 的髋部路径 + 30% 的腿部路径

这就是"权重混合/渐变"，让过渡区网格自然地"跟着两边走"。

参考模型（Purifier Inase 18 None）数据：
| 区域 | 下半身覆盖 | 足D.L覆盖 | 同时存在（过渡区） |
|------|-----------|---------|--------------|
| 参考模型 | 14,557 顶点 | 7,375 顶点 | **3,204 顶点** |
| 修复前当前模型 | 4,424 顶点 | 1,872 顶点 | **0 顶点** ⚠️ |

## 三、XPS 原始权重的特点

XPS 模型骨骼层级扁平，只有一条腿链：

```
root hips（≈下半身）
 └── spine lower（≈下半身）
      └── leg left thigh（≈左足）
           └── leg left knee（≈左ひざ）
```

XPS 的 `spine lower`（下半身）权重范围：
```
腰部   ████████████████████
髋部   ████████████████████  ← 有权重
腹股沟 ████████████          ← 少量权重
大腿根 ████████              ← 一点点
大腿中 ░░░░░░░░░░            ← 基本没了
```

`leg left thigh`（左足）权重范围：
```
腰部   ░░░░░░
腹股沟 ████              ← 从这里开始
大腿中 ██████████████    ← 主力区域
膝盖上 ██████████████
```

**关键：XPS 原始的过渡区是天然存在的**，`spine lower` 和 `leg left thigh` 在腹股沟区域本来就有重叠，这是正确的。

## 四、转换过程中的三个危险节点

### 危险节点 ①：步骤2「补全缺失骨骼」— 本身无问题

```
complete_missing_bones 创建 足D.L 时：
  足D.L 的权重 ← 直接复制自 左足（FK骨）的权重
```

此时状态：
- 下半身：腰部+髋部+腹股沟 都有权重 ✅
- 足D.L：腹股沟+大腿中+膝盖上 都有权重 ✅
- 两者在腹股沟区域重叠 ✅ 这是对的

**这一步本身没问题，过渡区是健康的。**

---

### 危险节点 ②：步骤7「修复非变形骨权重」— 问题的直接触发点

运行 `fix_nondeform_weights` 时（旧逻辑）：

```
左足 use_deform=False → 找最近可变形父骨
                                   ↓
旧代码：走父级链 → 左ひざ → 左足 → 下半身  ← 找到了！
         把 左足 的权重全部转移给 下半身
```

结果：大腿顶点也有了下半身权重（不该有的）。

**为了修这个问题，加了 cleanup 函数**，把凡是有足D 的顶点的下半身全清零——但：

```
清理的副作用：
  腹股沟区顶点：足D.L=0.3，下半身=0.7（天然混合）
               ↓ cleanup 看到有 足D，就清掉 下半身
  清理后：      足D.L=0.3，下半身=0.0  ← 顶点权重只剩 0.3！
```

**天然存在的过渡区被顺带摧毁。**

---

### 危险节点 ③：步骤8「填充缺失MMD骨权重」— 二次伤害

`fix_missing_weights` 向 `足D.L` 填充权重时，如覆盖不足会从祖先骨填充，
填充后 `足D.L` 和 `下半身` 再次重叠，cleanup 再次运行，过渡区被破坏两次。

---

## 五、整个问题链条

```
XPS原始           步骤7旧代码          cleanup           步骤8+cleanup
─────────         ─────────────       ─────────         ────────────────
下半身↔左足        左足权重→下半身       凡有足D→           再填充再清理
天然重叠     →     下半身暴涨      →    清下半身      →    过渡区彻底消失
（健康）           （问题产生）          （过激反应）        （二次伤害）
```

---

## 六、两个修复合力解决

### 修复1：`LEG_FK_TO_D` 映射表（`fix_nondeform_weights`）

FK腿骨直接映射到对应D系骨，不再走父级链找到下半身：

```python
LEG_FK_TO_D = {
    "左足":   "足D.L",   "右足":   "足D.R",
    "左ひざ": "ひざD.L", "右ひざ": "ひざD.R",
    "左足首": "足首D.L", "右足首": "足首D.R",
}
```

效果：步骤7不再错误地把左足权重注入下半身，不产生假冲突。

### 修复2：cleanup 阈值化（`_weight_cleanup_leg_torso_conflict`）

```python
# 旧代码（错误）：只要有一丁点足D就清下半身
has_d = any(g.weight > 0.001 for g in 足D系权重)
if has_d:
    下半身权重 = 0

# 新代码（正确）：只在足D占主导（≥0.6）时才清
D_DOMINANT_THRESHOLD = 0.6
d_total = sum(足D系各权重)
if d_total >= 0.6:   # 明确处于腿部区域
    下半身权重 = 0
# d_total < 0.6 → 过渡区，保留混合
```

效果：

```
修复前（硬切割）：
  下半身:  █████████████░░░░░  ← 截止在 Z=13.03
  足D.L:  ░░░░░░░████████████  ← 从 Z=13.03 开始
                ↑ 硬切割！

修复后（平滑过渡）：
  下半身:  ████████████▓▓▓░░░░  ← 权重渐渐变小
  足D.L:  ░░░░░░░▓▓▓████████  ← 权重渐渐变大
                ↑↑↑↑ 平滑过渡区
```

---

## 七、为什么阈值选 0.6？

参考模型的权重分布规律：
- 足D.L 权重 < 0.6 的顶点 → 大多分布在髋部/腹股沟 → 应和下半身共存
- 足D.L 权重 ≥ 0.6 的顶点 → 明确处于大腿中段以下 → 下半身留在这里才是真问题

0.6 是经验值，代表"已经超过大腿根部、进入大腿主体区域"的临界点。

---

## 八、一句话总结

> 腰臀交界处的顶点必须同时受两块骨骼影响，靠**权重比例**在两者之间插值，
> 才能让网格跟骨骼运动平滑过渡。旧代码把这个过渡区的混合全删掉了，
> 所以动起来有明显断裂感。
